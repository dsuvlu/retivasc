What happens to the data, step by step

This is rendered as HTML so the report remains readable, searchable, and paired with the code paths that perform each operation.

01

data/raw/fives/

Raw data landing zone

The official FIVES archive is unpacked locally. Fundus images and manual vessel masks remain outside version control.

src/retivasc/io.pyload_fives_manifest
  • raw images stay gitignored
  • official train/test folders retained
02

800 image-mask rows with label and official_split metadata

Manifest audit

Each fundus image is paired with its manual vessel mask. Dataset, image id, label, official split, and split group are normalized into a table.

src/retivasc/io.pyload_fives_manifest notebooks/02_fives_modeling_calibration_demo.pylocal data check
  • image-mask pairing validated
  • split metadata preserved
03

manual vessel mask arrays

Mask preprocessing

Mask PNGs are loaded, converted to grayscale, binarized, and downsampled to max dimension 512 for fast reproducible feature extraction.

src/retivasc/preprocess.pyensure_grayscale src/retivasc/preprocess.pyresize_mask_to_max_dim
  • no synthetic masks created
  • feature_max_dim recorded
04

one interpretable feature vector per image

Vascular feature extraction

The binary mask is skeletonized and summarized as density, skeleton length density, branchpoint density, fractal dimension, tortuosity, and connected components.

src/retivasc/features.pyextract_vascular_features src/retivasc/skeleton.pyskeletonize_mask
  • species-agnostic feature definitions
  • derived table only
05

data/interim/fives_features_max512.parquet

Feature cache

The derived feature table is cached so report regeneration does not re-read and reprocess every raw mask.

notebooks/02_fives_modeling_calibration_demo.pycache check pandas.to_parquet
  • 800 cached rows expected
  • cache includes feature_max_dim
06

600 train rows and 200 test rows

Leakage-aware split

The official FIVES image-level split is used when present. Split groups are checked for image-disjoint evaluation, but patient-disjoint splitting is not possible without public patient identifiers.

src/retivasc/splits.pyassert_group_split_safe sklearn Pipeline
  • official image-level split preferred
  • patient identifiers unavailable
07

figures/fives_calibration_demo.png, reports/fives_metrics.json

Modeling and calibration

A simple logistic model demonstrates evaluation discipline on FIVES disease-vs-normal labels using AUROC, AUPRC, Brier score, and bootstrap CIs.

src/retivasc/plotting.pyplot_calibration sklearn.metrics
  • calibration reported
  • not an ADRD prediction claim

reports/retivasc_pi_demo.html

Static PI report

The deliverable consumes derived figures, cached features, metric JSON, explicit caveats, and this HTML audit trail.

src/retivasc/report_html.pybuild_report
